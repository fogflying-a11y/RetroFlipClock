# 翻页时钟 Android APP — 产品需求文档 (PRD)

> 版本：v1.0 | 日期：2026-06-22 | 状态：初稿

---

## 一、产品概述

### 1.1 产品定位

一款复古拟物化的翻页时钟 Android 应用，利用闲置手机屏幕常亮充当电子翻页钟，还原经典机械翻页时钟的视觉与听觉体验。

### 1.2 目标用户

- 拥有闲置旧手机、希望将其改造为桌面时钟的用户
- 喜欢复古工业风、机械翻页质感的设计爱好者
- 需要床头/桌面时钟替代品的用户

### 1.3 核心价值主张

**"把闲置手机变成一件会翻页的机械艺术品。"**

通过高度还原机械翻页时钟的物理视觉动效、翻页音效与复古字体，让数字时钟拥有真实的"机械生命感"。

## 二、功能需求

### 2.1 核心功能 — 翻页时钟显示

| 编号 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| F-01 | 翻页动画效果 | P0 | 每位数字变化时触发从上至下的翻页动画，还原物理翻页时钟的视觉体验。动画需平滑、有重量感 |
| F-02 | 时:分 显示模式 | P0 | 默认显示格式，展示 HH:MM，共 4 个翻页卡片 |
| F-03 | 时:分:秒 显示模式 | P0 | 可选格式，展示 HH:MM:SS，共 6 个翻页卡片。秒数翻页频率高，需保证动画流畅不卡顿 |
| F-04 | 24 小时制 | P0 | 时间范围 00:00 ~ 23:59（或 00:00:00 ~ 23:59:59） |
| F-05 | 12 小时制 | P0 | 时间范围 12:00 AM ~ 11:59 PM，需在时钟区域旁显示 AM / PM 标识 |
| F-06 | 翻页声音开关 | P1 | 可选开启/关闭。开启后每位数字翻页时播放清脆的"嗒"机械翻页音效。声音大小可调（低/中/高） |

#### F-01 翻页动画详细规格

- **动画类型**：上半部分向下翻折至水平中线，随后下半部分同步翻转到位
- **动画时长**：单次翻页约 200ms ~ 350ms，需有轻微弹性缓动效果
- **阴影效果**：翻页过程中卡片产生投影，增强立体感
- **分隔符**：时与分之间、分与秒之间以冒号 `:` 分隔，冒号以闪烁方式呈现（每秒亮灭一次）
- **卡片样式**：深色圆角矩形背景 + 白色/米白色大号数字，参考附件截图风格

### 2.2 万年历显示功能

| 编号 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| F-07 | 万年历开关 | P1 | 可在设置中开启/关闭。开启后在时钟顶部区域显示日历信息 |
| F-08 | 公历日期显示 | P1 | 格式：`YYYY年MM月DD日`，如 `2026年06月22日` |
| F-09 | 农历日期显示 | P1 | 完整格式：`[干支纪年] [农历月] [农历日]`，如 `丙午年 五月初九` |
| F-10 | 农历数据源 | P1 | 基于内置农历算法库（如 `lunarcalendar` 或自定义天文算法），无需联网即可计算，保证离线可用 |
| F-11 | 日期变更翻页动效 | P1 | 当日期或农历数据发生变化时（跨日 00:00 时刻），日历区域的数字/文字需以与时间区域相同的翻页动画效果过渡 |

#### F-09 农历格式详细说明

```
公历：2026年06月22日
农历：丙午年 五月 初九
```

- **干支纪年**：根据天干（甲乙丙丁戊己庚辛壬癸）与地支（子丑寅卯辰巳午未申酉戌亥）组合，共 60 个循环
- **农历月**：正月、二月、...、腊月（十二月）
- **农历日**：初一、初二、...、三十
- **闰月处理**：遇闰月时标注为"闰X月"，如"闰二月"

### 2.3 系统设置

| 编号 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| F-12 | 屏幕常亮（Wake Lock） | P0 | 应用在前台运行时保持屏幕常亮，防止休眠。需提供"防止烧屏"选项（见 F-14） |
| F-13 | 12h / 24h 切换 | P0 | 设置项，切换后即时生效，无需重启应用 |
| F-14 | 防烧屏像素偏移 | P2 | 每隔一段时间（可设 1/5/10/30 分钟）将画面整体偏移 1~2 像素，防止 OLED 屏幕烧屏 |
| F-15 | 亮度调节 | P2 | 应用内可独立调节屏幕亮度（通过 WindowManager.LayoutParams），方便床头/夜间使用 |
| F-16 | 开机自启（可选） | P2 | 允许用户选择是否在设备开机后自动启动应用 |

### 2.4 字体与视觉风格

| 编号 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| F-17 | 复古翻页数字字体 | P0 | 使用专门的翻页时钟拟物化字体，确保数字造型厚重、有机械感。候选字体：DSEG7（七段数码管风格）、Ticking Timebomb、或自定义翻页数字字体 |
| F-18 | 深色背景主题 | P0 | 整体采用深黑/深灰背景（#1A1A1A ~ #2C2C2C），与白色/米白数字形成高对比度 |
| F-19 | 翻页卡片拟物化 | P0 | 每张数字卡片需具备：圆角、中间分割线（模拟翻页转轴）、轻微渐变/阴影以增强立体感 |
| F-20 | 字体大小自适应 | P1 | 根据屏幕尺寸和显示内容（4位 vs 6位数字 + AM/PM 标识）自动调整字体大小，保证画面居中且饱满 |

## 三、非功能需求

### 3.1 性能要求

| 指标 | 目标值 |
|------|--------|
| 冷启动时间 | ≤ 2 秒 |
| 帧率（翻页动画） | ≥ 55 FPS，无明显掉帧 |
| 内存占用 | ≤ 80 MB |
| 电池消耗（屏幕常亮模式） | 每小时 ≤ 5%（取决于屏幕亮度与硬件） |
| 秒级翻页无卡顿 | 在低端设备（2GB RAM）上也能流畅运行 |

### 3.2 兼容性要求

| 维度 | 要求 |
|------|------|
| Android 版本 | 最低支持 Android 8.0 (API 26)，推荐 Android 10+ |
| 屏幕适配 | 支持 4.7" ~ 7" 屏幕，横屏优先，兼容竖屏 |
| 分辨率 | 支持 720p / 1080p / 2K / 4K |
| 架构 | 支持 ARM64 为主，兼容 ARM32 |

### 3.3 离线能力

- **核心功能完全离线可用**：时钟显示、翻页动画、翻页音效、农历计算均不依赖网络
- 无需任何网络权限

### 3.4 安全性

- 不收集用户数据
- 不请求不必要的权限（仅请求：`WAKE_LOCK`、`RECEIVE_BOOT_COMPLETED`（可选）、`SYSTEM_ALERT_WINDOW`（悬浮窗模式，P2））
- 所有农历数据本地计算，不上传

## 五、技术方案建议

### 5.1 推荐技术栈

| 层次 | 推荐方案 |
|------|---------|
| 语言 | Kotlin |
| UI 框架 | Jetpack Compose |
| 动画 | Compose Animation API + Canvas 自定义翻页动画 |
| 农历计算 | 开源农历库（如 `cn.leo.calendar.lunar` 或自建天文算法） |
| 音频播放 | `MediaPlayer` 或 `SoundPool`（短音效推荐 SoundPool） |
| 屏幕常亮 | `WindowManager.LayoutParams.FLAG_KEEP_SCREEN_ON` |
| 像素偏移 | Compose `Modifier.offset()` + `LaunchedEffect` 定时触发 |
| 架构 | MVVM（单 Activity，无复杂业务逻辑，无需过度架构） |

### 5.2 翻页动画实现思路

```
状态机：
  IDLE ─(时间变更)──▶ FLIPPING_TOP ──(动画完成)──▶ FLIPPING_BOTTOM ──(动画完成)──▶ IDLE

FLIPPING_TOP:    上半张卡片以中轴线为旋转轴，从 0° 旋转至 90°（向下翻）
FLIPPING_BOTTOM: 下半张新卡片从 -90° 旋转至 0°（翻到位）
```

- 使用 Compose 的 `graphicsLayer` + `rotationX` 实现 3D 翻转效果
- 为每张卡片设置 `cameraDistance` 以控制透视感
- 翻页过程中使用 `Shadow` 或渐变遮罩增强立体感

### 5.3 农历算法说明

- 采用 1900 ~ 2100 年的农历数据表（约 3KB），存储在 `assets/` 目录
- 根据公历日期查表计算农历月日与干支年
- 干支年以立春为分界（非正月初一），需实现立春日期计算逻辑

## 八、风险与注意事项

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 翻页动画在低端设备掉帧 | 体验差 | 秒级翻页时降级动画精度，或提供"简化动画"选项 |
| 农历计算边界 case（闰月、跨闰年） | 日期错误 | 覆盖 1900~2100 年全量测试，与权威万年历逐日对比校验 |
| OLED 烧屏 | 硬件损伤 | 提供像素偏移功能，并在首次启动时提示用户 |
| 后台被系统杀进程 | 时钟停摆 | 使用前台 Service 保活，或依赖 AlarmManager 周期性唤醒 |
| 翻页音效版权 | 法律风险 | 自行录制或使用 CC0 授权的音效素材 |
