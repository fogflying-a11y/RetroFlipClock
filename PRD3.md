# RetroFlip 时钟 — Android APP 产品需求文档 (PRD)

> 版本：v1.0（正式版）| 日期：2026-06-22 | 状态：已确认 | 文档编号：PRD-RF-001

---

## 一、产品概述

### 1.1 产品信息

| 项目 | 说明 |
|------|------|
| 产品名称 | **RetroFlip 时钟** |
| 产品定位 | 一款专为闲置 Android 手机设计的全屏复古翻页时钟应用 |
| 核心价值 | 提供极致拟物化的翻页视觉与听觉体验，将废旧手机转化为高颜值的桌面电子时钟 |
| 价值主张 | **"把闲置手机变成一件会翻页的机械艺术品。"** |

### 1.2 目标用户

- 拥有闲置旧手机、希望将其改造为桌面时钟的用户
- 喜欢复古工业风、机械翻页质感的设计爱好者
- 需要床头/桌面时钟替代品的用户

## 二、功能需求

### 2.1 核心功能 — 翻页时钟显示

| 编号 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| F-01 | 翻页动画效果 | P0 | 每位数字变化时触发从上至下的翻页动画，还原物理翻页时钟的视觉体验。动画需平滑、有重量感 |
| F-02 | 时:分 显示模式 | P0 | 默认显示格式，展示 HH:MM，共 4 个翻页卡片 |
| F-03 | 时:分:秒 显示模式 | P0 | 可选格式，展示 HH:MM:SS，共 6 个翻页卡片。秒数翻页频率高，需保证动画流畅不卡顿 |
| F-04 | 24 小时制 | P0 | 时间范围 00:00 ~ 23:59（或 00:00:00 ~ 23:59:59） |
| F-05 | 12 小时制 | P0 | 时间范围 12:00 AM ~ 11:59 PM，需在时钟区域旁显示 AM / PM 标识 |
| F-06 | 翻页声音开关 | P1 | 可选开启/关闭。开启后每位数字翻页时播放清脆的"嗒"机械翻页音效。声音大小可调（低/中/高） |

#### F-01 翻页动画详细规格

- **动画类型**：上半部分卡片沿 X 轴（横轴）向前翻转向下至水平中线，随后下半部分同步翻转到位
- **动画时长**：单次翻页约 200ms ~ 350ms
- **物理曲线**：采用 Overshoot（回弹）动画曲线，翻转触底时有微小的回弹物理效果，模拟真实卡片惯性
- **阴影效果**：翻页过程中卡片产生动态阴影与高光变化，模拟现实中卡片翻动时遮挡光线的效果，增强 3D 立体感
- **分隔符**：时与分之间、分与秒之间以冒号 `:` 分隔，冒号以闪烁方式呈现（每秒亮灭一次）
- **卡片样式**：深色圆角矩形背景 + 白色/米白色大号数字，中间水平分割线模拟翻页转轴
- **音效连贯性**：当开启秒数显示时，连续翻页音效需处理好连贯性，避免音效重叠造成的嘈杂感，SoundPool 并发通道需做限流

### 2.2 万年历显示功能

| 编号 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| F-07 | 万年历开关 | P1 | 可在设置中开启/关闭。开启后在时钟顶部区域显示日历信息 |
| F-08 | 公历日期显示 | P1 | 格式：`YYYY年MM月DD日`，如 `2026年06月22日` |
| F-09 | 农历日期显示 | P1 | 完整格式：`[干支纪年] [农历月] [农历日]`，如 `丙午年 五月初九` |
| F-10 | 农历数据源 | P1 | 基于内置农历算法库，无需联网即可计算，保证完全离线可用 |
| F-11 | 日期变更翻页动效 | P1 | 当日期或农历数据发生变化时（跨日 00:00 时刻），日历区域的数字/文字需以与时间区域相同的翻页动画效果过渡 |

#### F-09 农历格式详细说明

```
公历：2026年06月22日
农历：丙午年 五月 初九
```

- **干支纪年**：根据天干（甲乙丙丁戊己庚辛壬癸）与地支（子丑寅卯辰巳午未申酉戌亥）组合，共 60 个循环。以**立春**为分界（非正月初一）
- **农历月**：正月、二月、...、腊月（十二月）
- **农历日**：初一、初二、...、三十
- **闰月处理**：遇闰月时标注为"闰X月"，如"闰二月"
- **数据范围**：内置 1900 ~ 2100 年农历数据表（约 3KB），存储在 `assets/` 目录

### 2.4 字体与视觉风格

| 编号 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| F-17 | 复古翻页数字字体 | P0 | 使用专门的翻页时钟拟物化字体，确保数字造型厚重、有机械感。首选字体：**DSEG7**（七段数码管风格）或 **Ticking Timebomb**；备选经典无衬线字体：**Helvetica / Bebas Neue**，需具有轻微的磨砂质感或厚度感 |

## 三、非功能需求

### 3.1 性能要求

| 指标 | 目标值 |
|------|--------|
| 冷启动时间 | ≤ 2 秒 |
| 帧率（翻页动画） | ≥ 55 FPS，无明显掉帧 |
| 内存占用 | ≤ 80 MB |
| 电池消耗（屏幕常亮模式） | 每小时 ≤ 5%（取决于屏幕亮度与硬件） |
| 秒级翻页无卡顿 | 在低端设备（2GB RAM）上也能流畅运行 |

### 3.2 兼容性要求

| 维度 | 要求 |
|------|------|
| Android 版本 | 最低支持 Android 8.0 (API 26)，推荐 Android 10+ |
| 屏幕适配 | 支持 4.7" ~ 7" 屏幕，横屏优先，兼容竖屏 |
| 分辨率 | 支持 720p / 1080p / 2K / 4K |
| 架构 | 支持 ARM64 为主，兼容 ARM32 |

### 3.3 离线能力

- **核心功能完全离线可用**：时钟显示、翻页动画、翻页音效、农历计算均不依赖网络
- 无需任何网络权限

### 3.4 安全性

- 不收集用户数据
- 不请求不必要的权限（仅请求：`WAKE_LOCK`、`RECEIVE_BOOT_COMPLETED`（可选）、`SYSTEM_ALERT_WINDOW`（悬浮窗模式，P2））
- 所有农历数据本地计算，不上传

## 五、技术方案

### 5.1 推荐技术栈

| 层次 | 推荐方案 | 选型理由 |
|------|---------|---------|
| 语言 | Kotlin | Android 官方首选，Compose 原生支持 |
| UI 框架 | Jetpack Compose | 声明式 UI，动画 API 强大，适合自定义翻页效果 |
| 动画 | Compose Animation API + Canvas 自定义翻页动画 | `graphicsLayer` + `rotationX` 实现 3D 翻转 |
| 农历计算 | 内置农历数据表（assets）+ 自建算法 | 完全离线，无第三方依赖风险 |
| 音频播放 | SoundPool | 短音效延迟低，适合高频秒级翻页场景 |
| 屏幕常亮 | `WindowManager.LayoutParams.FLAG_KEEP_SCREEN_ON` | 标准方案，无需额外权限 |
| 像素偏移 | Compose `Modifier.offset()` + `LaunchedEffect` 定时触发 | 简单可靠 |
| 架构 | MVVM（单 Activity） | 业务逻辑简单，无需过度架构 |
| 数据存储 | DataStore (Preferences) | 保存用户设置（制式、音效、亮度等） |

### 5.2 翻页动画实现方案

#### 状态机设计

```
IDLE ─(时间变更)──▶ FLIPPING_TOP ──(动画完成)──▶ FLIPPING_BOTTOM ──(动画完成)──▶ IDLE

FLIPPING_TOP:    上半张卡片以中轴线为旋转轴，rotationX 从 0° 旋转至 90°（向下翻）
FLIPPING_BOTTOM: 下半张新卡片 rotationX 从 -90° 旋转至 0°（翻到位）
```

#### 实现要点

- 使用 Compose `graphicsLayer` + `rotationX` 实现 3D 翻转效果
- 设置 `cameraDistance` 控制透视感，建议 800~1200 范围
- 翻页过程中使用 Canvas 绘制渐变遮罩模拟动态阴影与高光
- 动画曲线使用 `OvershootInterpolator`（回弹效果），参数建议 tension = 1.5
- **音效同步**：在 FLIPPING_TOP 动画开始时触发 SoundPool 播放 Tick 音效
- **音效限流**：秒级翻页场景下，SoundPool 并发通道上限 4，超出时丢弃旧音效

### 5.3 农历算法方案

- 采用 1900 ~ 2100 年的农历数据表（约 3KB），存储在 `assets/lunar_data.json`
- 数据表字段：每年农历月的天数（大月30天/小月29天）、闰月位置、春节公历日期
- 根据公历日期查表计算农历月日与干支年
- 干支年以**立春**为分界（非正月初一），需实现立春日期计算逻辑
- 验收标准：与权威万年历（如中国科学院紫金山天文台数据）逐日对比校验

## 八、风险与缓解措施

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 翻页动画在低端设备掉帧 | 体验差 | 秒级翻页时降级动画精度，或提供"简化动画"选项 |
| 农历计算边界 case（闰月、跨闰年、立春分界） | 日期错误 | 覆盖 1900~2100 年全量测试，与权威万年历逐日对比校验 |
| OLED 烧屏 | 硬件损伤 | 提供像素偏移功能，并在首次启动时提示用户 |
| 后台被系统杀进程 | 时钟停摆 | 使用前台 Service 保活，或依赖 AlarmManager 周期性唤醒 |
| 翻页音效版权 | 法律风险 | 自行录制或使用 CC0 授权的音效素材 |
| 秒级音效资源占用过高 | 卡顿/耗电 | SoundPool 并发通道限流，超出丢弃旧音效；提供音效音量分级 |

## 九、术语表

| 术语 | 说明 |
|------|------|
| 翻页动画 | Flip Animation，模拟物理翻页时钟的卡片翻转动效 |
| Overshoot 曲线 | 动画缓动曲线，运动到终点时略微超出后回弹，模拟物理惯性 |
| Tick/Clack | 机械翻页时钟翻页时发出的清脆""声 |
| 干支纪年 | 中国传统历法中的天干地支纪年，60年一循环，以立春为分界 |
| 烧屏 | OLED 屏幕长时间显示静态图像导致的永久性残影 |
| Wake Lock | Android 保持屏幕/系统唤醒的机制 |
| SoundPool | Android 短音效播放引擎，低延迟，适合高频音效场景 |
